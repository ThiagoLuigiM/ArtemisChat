# Expressões e padrões a evitar

> A IA deve **nunca** produzir os padrões abaixo. Adicione novos itens conforme
> for revisando devolutivas geradas e identificar vícios.

## Gerúndio burocrático
- ~~"Estaremos providenciando"~~ → "Providenciaremos"
- ~~"Iremos verificar"~~ → "Verificaremos"
- ~~"Estaremos enviando"~~ → "Enviaremos" (ou melhor: já enviei)
- ~~"Vamos estar disponibilizando"~~ → "Disponibilizaremos"

## Justificativas excessivas
- ~~"Conforme análise realizada pela equipe de desenvolvimento"~~ → desnecessário
- ~~"Após exaustivos testes"~~ → desnecessário
- ~~"Devido à complexidade do cenário"~~ → desnecessário

## Pedidos de desculpa
- ~~"Pedimos desculpas pelo transtorno"~~ → não cabe na devolutiva técnica
- ~~"Lamentamos o ocorrido"~~ → não cabe

## Vaguezas
- ~~"Realizamos os devidos ajustes"~~ → diga **o que** foi ajustado
- ~~"O sistema foi otimizado"~~ → diga **o que** foi otimizado
- ~~"Foi feita uma melhoria"~~ → diga **qual** melhoria

## Anglicismos desnecessários
- ~~"Deployar"~~ → "Liberar"
- ~~"Hotfix"~~ → "Correção emergencial" (a menos que o contexto exija o termo técnico)
- ~~"Patch"~~ → "Correção" (a menos que seja patch oficial nomeado)

## Construções pleonásticas
- ~~"Encaminhamos para análise dos analistas"~~
- ~~"Verificação será verificada"~~
- ~~"Validação após validar com o cliente"~~
